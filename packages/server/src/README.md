# Server 模块架构说明

## 项目结构

```
server/
├── config.js      # 配置管理（环境变量、路径等）
├── courses.js     # 课程扫描和管理
├── data.js        # 文件夹数据管理
├── proxy.js       # CDN 代理和缓存
├── routes.js      # API 路由
├── socket.js      # Socket.io 实时通信
├── submissions.js # 学生提交和座位表管理
└── utils.js       # 工具函数
```

## 模块说明

### 1. config.js - 配置管理
- 集中管理所有配置项
- 端口、路径、超时等配置
- 目录初始化
- 环境变量处理

### 2. courses.js - 课程管理
- 扫描课程目录
- 课程元数据提取
- 课程去重
- 课程删除

### 3. data.js - 文件夹数据管理
- 文件夹 CRUD 操作
- 课件与文件夹关联
- 文件夹移动/重命名
- 数据持久化

### 4. proxy.js - CDN 代理
- 静态资源代理
- 智能缓存
- 重定向处理
- 超时控制

### 5. routes.js - API 路由
- RESTful API 定义
- 请求验证
- 响应格式化
- 状态管理

### 6. socket.js - 实时通信
- Socket.io 连接管理
- 事件处理
- 标注同步
- 学生管理

### 7. submissions.js - 学生提交
- 提交数据保存
- 座位表管理
- 学生信息查询
- 文件合并/分离

### 8. utils.js - 工具函数
- 文件读取
- 数据加载
- 通用工具

## 主入口 (server.js)

主入口文件负责：
1. 初始化 Express 和 HTTP 服务器
2. 配置中间件
3. 注册路由
4. 启动 Socket.io
5. 启动服务器

## 数据流

```
客户端请求 → Express 中间件 → API 路由 → 业务模块 → 数据层
                ↓                    ↓
           代理处理            Socket.io
                ↓
           CDN/文件系统
```

## 扩展指南

### 添加新的 API 端点

1. 在 `routes.js` 中添加路由
2. 在对应的业务模块中实现逻辑
3. 在 `config.js` 中添加必要的配置

### 添加新的 Socket 事件

1. 在 `socket.js` 中添加事件监听
2. 实现事件处理逻辑
3. 使用 `io.emit()` 广播事件

### 添加新的代理路由

1. 在 `proxy.js` 中添加处理函数
2. 在 `server.js` 中注册路由
3. 测试 CDN 下载和缓存

## 注意事项

1. **路径处理**: 使用 `path.join()` 和 `path.resolve()` 确保跨平台兼容
2. **错误处理**: 所有异步操作都要有 try-catch
3. **日志记录**: 使用 console.log 记录关键操作
4. **配置管理**: 避免硬编码，使用 `config.js`
5. **模块导出**: 只导出必要的函数和变量

## 性能优化

1. **缓存策略**: 使用内存缓存减少文件 I/O
2. **CDN 代理**: 静态资源从 CDN 下载并缓存
3. **连接池**: 复用 HTTP 连接
4. **流式处理**: 大文件使用流式传输

## 安全考虑

1. **输入验证**: 验证所有用户输入
2. **路径安全**: 防止路径遍历攻击
3. **速率限制**: 考虑添加速率限制中间件
4. **HTTPS**: 生产环境使用 HTTPS
