# **LumeSync (萤火课堂) 参赛作品全套申报文档**

## **第一部分：作品简介**

**作品名称**：LumeSync (萤火课堂)——轻量化局域网 AI 互动教学平台

### **1\. 设计初衷**

在传统机房教学中，多数系统依赖“屏幕共享”或“视频广播”，导致学生参与感低、互动性差，且教师难以直观呈现抽象逻辑。然而，开发高质量的互动课件通常需要深厚的编程基础，这为一线教师构筑了极高的技术门槛。

LumeSync 应运而生。我们致力于打破“屏幕共享”的僵局，通过“AI 辅助创作 \+ 本地数据同步”的全新架构，让老师能零门槛创作互动课件，让学生在同步实验中深度沉浸，点亮每一个探究的微光。

### **2\. 作品概述**

LumeSync 是一款基于 Node.js 和 Electron 构建的智能化局域网同步教学系统。它不仅是一个实时同步的互动框架，更是一个集成了 AI 赋能能力的教学生态。系统通过内置的 AI 课件创作 Skill，配合轻量级数据指令驱动，实现了教师端全局控制、AI 快速建模与学生端自主交互的完美统一。

### **3\. 核心功能**

* **AI 零门槛课件创作**：系统内置专属 AI Skill（指令集）。教师只需通过自然语言描述教学需求，即可利用 AI 辅助生成基于 .tsx 格式的深度互动课件，实现秒级创意转化。  
* **深度交互同步引擎**：抛弃传统的像素广播，采用数据指令同步机制。教师端操作即时触发学生端本地渲染，确保进度统一的同时，保留了动态图表、算法模拟等内容的实时交互活性。  
* **可视化机房管理**：提供座位网格与列表双视图，支持 CSV 批量导入座位布局，实时监控学生在线状态与学习进度。  
* **系统级安全防护**：学生端封装为受保护的 Windows 系统服务，支持管理员密码验证卸载，有效防止教学过程中的非正常退出。

## **第二部分：开源代码与组件使用情况说明**

**作品版本**：v2.0

**开源协议**：MIT License

### **1\. 核心基础架构（本地集成）**

* **Electron (v29.0.0)**：跨平台桌面端容器，处理系统级交互。  
* **Express (v4.18.2)**：后端 Web 服务器及资源中控枢纽。  
* **Socket.io (v4.7.2)**：毫秒级状态同步核心引擎。

### **2\. 资源获取与智能缓存机制**

本项目设计了创新的**智能 CDN 缓存代理系统**。系统不预先集成体积庞大的前端依赖，而是通过后端拦截请求：

* **动态获取**：首次使用时从公网（jsDelivr, unpkg 等）下载资源。  
* **持久化本地化**：资源下载后自动存至局域网服务器 /lib 目录。  
* **离线运行**：后续请求完全由本地分发，确保在物理隔离的机房环境下稳定运行。

### **3\. 通过缓存机制使用的第三方库**

* **UI 与引擎**：React (v18), Tailwind CSS, Babel Standalone (JSX 热编译), FontAwesome。  
* **教学扩展**：Chart.js (图表绘制), KaTeX (公式渲染), Prism.js (代码高亮), face-api.js (人脸识别扩展)。

## **第三部分：作品安装说明**

### **1\. 环境要求**

* **操作系统**：Windows 10/11 (x64)。  
* **网络**：局域网连接，首次运行需互联网以完成资源缓存。

### **2\. 用户端安装**

* **教师端**：运行 LumeSync-教师端-Setup.exe。启动后自动开启局域网同步服务。  
* **学生端**：需以管理员权限运行 LumeSync-学生端-Setup.exe。安装后自动注册为 Windows 服务并开机自启。卸载需输入管理员密码（默认：admin123）。

### **3\. 源码部署（开发环境）**

1. 执行 npm install 安装基础依赖。  
2. 启动服务器：node server.js。  
3. 运行应用：npm run start:teacher 或 npm run start:student。

## **第四部分：设计思路**

### **1\. 时代背景与 AI 落地**

随着人工智能的高速发展，将 AI 辅助教学真正落地到一线课堂至关重要。LumeSync 的研发初衷即是利用 AI 降低互动教学的门槛，将“AI 辅助”从口号转化为“自然语言生成互动课件”的实际生产力。

### **2\. 范式演进：从像素到指令**

系统核心围绕“打破僵化广播”展开。传统视频广播（像素传输）无法参与交互且带宽压力大；LumeSync 采用“状态同步 \+ 本地渲染”，服务器仅传输状态指令，学生端本地引擎实时生成内容，将学生从“看客”转变为“参与者”。

### **3\. 品牌寓意：LumeSync 的微光意向**

名称取自拉丁语“Lume”（光）与“Sync”（同步）。寓意机房中每一台学生终端都是一个具备独立渲染能力的“发光体”。通过教师端的微光指引，全场终端实现协同律动，汇聚成照亮智慧课堂的星海。

## **第五部分：设计重点与难点**

### **1\. 设计重点**

* **指令驱动同步**：建立轻量化状态机模型，确保 1:60 规模下毫秒级响应。  
* **AI Skill 生态**：设计针对系统框架优化的 Prompt 工程，使 AI 生成的代码能直接通过 JSX 热编译运行。  
* **互动与秩序平衡**：在提供高自由度交互的同时，通过系统级服务确保教学秩序不被干扰。

### **2\. 设计难点**

* **主动隔离环境下的资源分发**：为保障课堂纪律，机房通常物理隔绝公网。我们攻克了 CDN 代理技术，实现了一次联网、永久离线可用的智能资源管理。  
* **学生端防卸载与常驻保护**：针对学生通过任务管理器关闭监控的行为，开发了基于 node-windows 的系统服务保护机制，并绑定了基于 NSIS 的密码校验流程。  
* **大规模并发性能优化**：通过异步资源加载器与 React 虚拟 DOM 机制，解决了热编译重型 AI 库时可能出现的界面卡顿问题。
