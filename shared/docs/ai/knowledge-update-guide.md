# 知识库更新指南

本文档说明如何为萤火课件系统添加新功能时更新知识库。

## 更新原则

每次添加新功能时，必须同步更新知识库，确保AI编辑器能够检索到相关知识。

## 更新流程

### 1. 确定更新内容

当添加新功能时，需要添加以下知识内容：

#### 功能相关
- 功能概述和使用方法
- 配置参数说明
- 使用示例
- 注意事项

#### API相关
- API接口说明
- 参数和返回值
- 代码示例
- 使用场景

#### 组件相关
- 组件功能介绍
- Props配置
- 使用示例
- 最佳实践

### 2. 选择分类

根据功能类型选择合适的分类：

| 功能类型 | 对应分类 | 文件 |
|---------|---------|------|
| 系统API | 系统API | `system-api.js` |
| 新增组件 | 互动组件 | `interactive-components.js` |
| 样式相关 | 样式系统 | `styling.js` |
| 性能优化 | 最佳实践 | `best-practices.js` |
| 教学应用 | 教学策略 | `teaching-strategies.js` |

### 3. 编写知识内容

使用以下模板：

```javascript
{
    id: 'unique-id',
    title: '功能标题',
    category: '分类名称',
    tags: ['功能名', '英文术语', '相关关键词'],
    content: `## 功能概述

简要描述功能的作用和用途。

## 使用方法

### 基本用法
\`\`\`tsx
// 代码示例
\`\`\`

### 配置说明
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|

## 注意事项
1. 注意事项1
2. 注意事项2

## 完整示例
\`\`\`tsx
// 完整示例代码
\`\`\`
`,
    isBuiltin: true
}
```

## 实际案例：添加投票组件

### 步骤1：添加组件知识

在 `interactive-components.js` 中添加：

```javascript
{
    id: 'vote-component',
    title: '投票组件 (VoteSlide)',
    category: '互动组件',
    tags: ['投票', 'VoteSlide', '实时统计', '课堂互动'],
    content: `## 功能概述

VoteSlide 是一个通用的实时投票组件...

## 使用方法
// ...使用说明
`,
    isBuiltin: true
}
```

### 步骤2：添加API知识

在 `system-api.js` 中添加：

```javascript
{
    id: 'built-in-components',
    title: '内置组件库',
    category: '系统API',
    tags: ['组件', 'VoteSlide', 'SurveySlide', 'WebPageSlide'],
    content: `## 可用组件

`,
    isBuiltin: true
}
```

### 步骤3：更新最佳实践

在 `best-practices.js` 中添加相关实践：

```javascript
{
    id: 'component-reusability',
    title: '组件复用最佳实践',
    category: '最佳实践',
    tags: ['组件', '复用', '模块化'],
    content: `// ...复用组件的建议
`,
    isBuiltin: true
}
```

## 检查清单

添加新功能后，确保完成以下检查：

- [ ] 在对应分类中添加功能说明
- [ ] 包含使用示例代码
- [ ] 添加配置参数说明
- [ ] 提供注意事项
- [ ] 如果是组件，在system-api中添加组件库说明
- [ ] 更新API文档（如果有新增API）
- [ ] 创建或更新功能使用指南文档
- [ ] 测试AI编辑器能否检索到新知识

## 维护规范

### 版本更新

每次功能更新时：
1. 更新知识库内容
2. 更新API文档
3. 更新功能指南
4. 添加变更日志

### 质量保证

- **准确性**：确保代码示例可运行
- **完整性**：涵盖所有重要功能点
- **易用性**：提供清晰的步骤说明
- **时效性**：及时反映最新功能

### 标签管理

知识标签应包含：
- 功能名称（中文）
- 英文术语
- 相关关键词
- 使用场景

示例：`['投票', 'VoteSlide', '实时统计', '课堂互动']`

## 测试验证

添加知识后进行测试：

### 1. 搜索测试
- 使用不同关键词搜索
- 验证搜索结果相关性
- 检查排序是否合理

### 2. 内容测试
- 验证Markdown渲染
- 检查代码块显示
- 确认链接有效

### 3. AI检索测试
- 在编辑器中测试AI能否检索到知识
- 验证AI生成的课件是否使用了新功能
- 检查生成的代码是否正确

## 常见问题

### Q: 功能更新后知识库多久生效？
A: 知识库在页面加载时自动加载，刷新页面即可生效。

### Q: 如何判断应该添加到哪个分类？
A: 参考本文档"选择分类"部分的表格，根据功能类型选择。

### Q: 知识内容可以使用哪些格式？
A: 支持纯文本和Markdown格式，包括标题、列表、代码块、链接等。

### Q: 如何确保知识能被AI正确检索？
A:
1. 选择合适的关键词标签
2. 在内容中重复重要关键词
3. 提供清晰的标题和概述
4. 包含完整的使用示例

1. ✅ 功能被准确记录
2. ✅ 使用方法清晰明了
3. ✅ 示例代码可直接运行
