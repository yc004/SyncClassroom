# 02 启动模式

## 1. `start:teacher`

用途：

- 启动教师端桌面程序
- 自动拉起教师端服务
- 进入完整课堂控制模式

适用：

- 正式上课
- 教师端本地联调

## 2. `start:teacher-server`

用途：

- 只启动教师端服务
- 不启动教师端 Electron 桌面界面

适用：

- API 联调
- 服务侧调试

## 3. `start:student`

用途：

- 启动学生端桌面程序
- 连接教师端地址并跟随课堂状态

适用：

- 学生机实际运行
- 学生端渲染与交互调试

## 4. `start:core`

用途：

- 启动独立运行时内核服务

适用：

- 运行时协议调试
- 内核仓独立调试
- 为后续真正独立部署做准备

## 5. 当前推荐的课堂部署方式

当前代码实际主链路是：

- 教师端负责课堂服务
- 学生端连接教师端
- `core` 独立运行主要用于内核调试和协议演进
