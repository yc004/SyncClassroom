# 02 API 变更记录

本页只记录当前架构下值得关心的接口边界变化。

## 当前稳定边界

- 教师端负责数据面
- 核心负责运行时
- 学生端不直接依赖核心数据面接口
- 插件只消费渲染能力，不参与正式课堂数据面

## 已完成的关键变更

### 1. 文件管理从核心迁出

当前课程扫描、目录管理、布局保存、提交落盘都归教师端负责。

### 2. 核心数据面接口已废弃

核心保留运行时服务，但以下数据面接口已经不再承载真实能力：

- `/api/courses`
- `/api/course-status`
- `/api/refresh-courses`
- `/api/components-manifest`

### 3. 学生端访问路径调整

学生端当前直接连接教师端服务，消费教师端提供的资源地址和课堂同步状态。

### 4. 课件开发入口调整

课件开发现在以 SDK 为中心，正式文档不再要求回查旧模板或旧 API 文档。

## 当前建议

1. 新课件直接按 [课件 SDK 参考](../05-course-development/04-course-sdk.md) 编写
2. 新服务对接以教师端数据面 API 为准
3. 新的课堂同步能力以运行时事件协议为准
