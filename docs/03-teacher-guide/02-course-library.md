# 02 课件与文件管理

## 1. 设计原则

课件与文件管理现在全部归教师端负责。这包括：

- 课程扫描
- 课件导入与导出
- 目录分类
- 组件清单
- 提交文件管理
- 课件静态资源托管

## 2. 课程相关能力

教师端服务提供以下课程管理接口：

- `/api/courses`
- `/api/course-status`
- `/api/refresh-courses`
- `/api/components-manifest`
- `/api/course-folders/*`

## 3. 导入与导出

当前教师端支持的课件格式包括：

- `.lume`
- 历史兼容格式：`.tsx`、`.ts`、`.jsx`、`.js`
- `.pdf`

## 4. 目录管理

教师端支持：

- 创建目录
- 删除目录
- 重命名目录
- 将课件移动到目录

目录管理的意义是让老师可以按学科、年级、章节组织课件库。

## 5. 学生提交管理

教师端负责：

- 保存学生提交
- 查看学生提交列表
- 浏览单个提交文件
- 批量打包下载
- 配置提交目录

## 6. 布局与座位表

教师端也负责：

- 获取当前布局
- 保存布局
- 在学生提交时按 IP 匹配学生信息
