# 01 系统介绍

## 1. 系统定位

萤火课堂是一套面向局域网教室场景的课堂同步系统。它把“教师控制、课件内容、学生展示、课件编辑”拆成独立模块，以降低耦合、提升维护性，并支持后续独立发布。

系统面向的典型场景包括：

- 机房授课
- 局域网同步演示
- 课件翻页与课堂互动
- 学生端统一展示与提交
- 本地课件编辑与预览

## 2. 系统由什么组成

系统当前由 4 个真实子仓库组成：

- `core`：运行时内核，负责连接控制、课堂状态同步、Socket 事件总线和渲染运行时。
- `teacher`：教师端，负责课件和文件管理、教师桌面应用、教师端服务、课堂控制。
- `student`：学生端，负责连接教师端、展示同步内容、参与互动和提交结果。
- `editor-plugin`：编辑器插件，负责本地 `.lume` 课件预览与编辑辅助。

## 3. 设计原则

- 教师端持有内容真源：课程目录、课件、布局、提交文件都由教师端管理。
- 核心只做运行时：核心不再承担课程扫描、文件落盘和业务目录管理。
- 学生端只消费状态与资源：学生端不直接承担数据管理职责。
- 编辑器插件保持轻量：插件只负责本地预览，不承担正式课堂职责。

## 4. 你应该如何理解这套系统

如果把整个系统看成一套专业设备：

- `teacher` 像控制台和内容服务器
- `student` 像教室终端
- `core` 像运行时控制内核
- `editor-plugin` 像课件编辑器和本地预览器
