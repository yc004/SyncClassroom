# 归档资料说明

本目录存放历史文档、迁移阶段文档和旧目录结构下的资料副本。

这些资料保留的目的主要有三个：

1. 保留项目演进历史
2. 方便对照旧路径和旧接口
3. 为仍依赖旧文档命名的脚本或说明提供过渡参考

## 归档分类

- `dev/`：旧版开发文档、迁移记录、拆仓阶段资料
- `user/`：旧版教师端、学生端和课堂视图使用说明
- `components/`：旧版组件开发文档
- `interaction/`：旧版互动同步文档
- `ai/`：旧版知识库与 AI 相关说明
- `project/`：项目申报和比赛材料

## 使用原则

- 需要了解当前系统时，优先阅读上一级正式文档
- 只有在排查历史实现、兼容旧流程或补齐项目演进记录时，再查阅归档区

正式文档入口：

- [文档总览](../README.md)
